    Karten

    Eine mit Karten gefüllte Rasteransicht im Inhaltsbereich wird mit einem <ul"> Element eingeleitet. Jede Karte ist ein <li> Element innerhalb dieses Elternelements.

    Karten können einen Link enthalten. Die anklickbare Fläche des Links erstreckt sich über die gesamte Karte. Das Linkelement kann einen Text enthalten, muss es aber nicht.

    <main>
  <ul>
    <li>
      <img src="/static/placeholder.svg"/>
      <h2>Kartenansicht</h2>
      <p>Vorlage für eine Ansicht, die Karten in einer Rasteransicht darstellt.</p>
    </li>
  </ul>
</main>

    • Übersicht Karten

      Vorlage für eine Übersicht, die Elemente als Karten präsentiert.

    • Übersicht Listen

      Vorlage für eine Übersicht, die Elemente als Liste darstellt.

    • Detailansicht

      Vorlage für die Detailansicht von Daten eines einzelnen Elements.

    • Formular

      Vorlage für die Bearbeitung von Daten in einem Formular.

      Vorlagen für Navigationselemente

    • Scannen

      Vorlage für eine Seite auf der ein Bild mit der Kamera aufgenommen werden kann.

      Vorlage für eine leere Seite, falls keine Elemente für die Anzeige vorhanden sind.

    Listen

    Ähnlich der Kartenansicht liefert die Listenansicht eine Übersicht gleichartiger Elemente. Jedes Element erstreckt sich über die gesamte Breite des Inhaltsbereichs und alle Elemente werden untereinander angezeigt.

    Elemente in der Listenansicht sind Tabellenzeilen (<tr>) in einer Tabelle (<table>). Der Haupttext kommt in die erste Spalte oder in die zweite Spalte, wenn zu dem Element ein Bild angezeigt werden soll. Das Bild kommt dann in die erste Spalte und die Überschrift des Haupttextes erstreckt sich dann über die ersten beiden Spalten.

    Alle weiteren Daten und Steuerelemente erscheinen ab der zweiten bzw. dritten Spalte

    Detailansicht

    Dies ist eine Blaupause für die Anzeige von detaillierten Daten eines einzelnen Elements.

    Die angezeigten Elemente werden zentriert im Inhaltsbereich ausgegeben und nehmen nicht die volle Breite des Inhaltsbereichs ein.

    Im einfachsten Fall werden die Daten als Description-List dargestellt. Dabei können Daten in Gruppen zusammengefasst werden, wenn die Gruppen jeweils eine eigene Description-List besitzen. Die Gruppen können eine Überschrift mit einem <h1> bis <h6> Element bekommen. Die Überschrift erfolgt aber in der Regel im Zentralbereich des Headers. 

    <dl>
  <dt>Name</dt>
  <dd>Paprika</dd>
  <dt>Anzahl</dt>
  <dd>8</dd>
  <dt>Änderung</dt>
  <dd><time>2025-02-01T17:06</time></dd>
</dl>

    Trennlinien

    Nach einer Gruppe von Daten, die sich in der selben Description-List befinden, kann die folgende Zeilen mit einer Linie getrennt werden. Dazu müssen die Paare von dt und dd Elementen von einem div Element umgeben werden.

    Name
    Rote Paprika
    Allgemeiner Name
    Paprika
    Anzahl
    6

    Mehrere Definitionsterme für einen Wert

    Für einen Wert können in der Description-List mehrere Definitionsterme angegeben werden. Dabei wird der erste Term hervorgehoben und die nachfolgenden Terme unterhalb des ersten Terms angeordnet, während der darzustellende Wert als Description-Details Element vertikal zentriert auf der linken Seite erscheint.

    Formulare

    Formulare werden mit einem form Element umschlossen. Das label umschließt das dazugehörige input Element. Damit kann auf das Attribute for für das Label und id für das Eingabeelemente verzichtet werden.

    Das form Element ist direktes Kind des main Elements. Es zentriert das Formular im Inhaltsbereich und beschränkt die Breite.

    Eingabefelder werden durch fieldset Elemente gruppiert.

    Ich möchte, dass Formulare aussehen wie in Gnome Anwendungen. Solange das Eingabeelement nicht den Fokus hält, erscheint der Inhalt des Eingabefeldes ohne Chrome.

    Buttons

    Primäre Steuerelemente werden von einem em Element umschlossen.

    Sekundäre Steuerlemente werden nicht besonders ausgezeichnet. Das führt jedoch zu einer Ungereimtheit mit Verknüpfungen, die als Schaltfläche dargestellt werden sollen.

    Steuerelemente und Verknüpfungen mit destruktivem Character werden von einem strong Element umschlossen. Allerdings sollten Verküpfungen keine Destruktiven Aktionen durchführen. Das gehört sich einfach nicht.

    Menüs

    Menüs können an unterschiedlichen Stellen verwendet werden. Im Header als Anwendungsmenü, in der Seitenleiste zur Navigation oder im Inhaltsbereich.

    Menüs in der Kopfzeile (header)

    Menüs in der Kopfzeile dienen Navigation, können aber auch Elemente enthalten, mit denen die aktuelle Ansicht konfiguriert wird und zum Beispiel zwischen Listen- und Rasteransicht oder die Sortierung umgeschaltet.
