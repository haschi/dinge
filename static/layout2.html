    Karten

    Eine mit Karten gefüllte Rasteransicht im Inhaltsbereich wird mit einem <ul> Element eingeleitet als direktes Kind des <main> Elements. Jede Karte ist ein <li> Element innerhalb dieses Elternelements.

    Karten können einen Link enthalten. Die anklickbare Fläche des Links erstreckt sich über die gesamte Karte. Das Linkelement kann einen Text enthalten, muss es aber nicht.

    Listen

    Ähnlich der Kartenansicht liefert die Listenansicht eine Übersicht gleichartiger Elemente. Jedes Element erstreckt sich über die gesamte Breite des Inhaltsbereichs und alle Elemente werden untereinander angezeigt.

    Elemente in der Listenansicht sind Tabellenzeilen (<tr>) in einer Tabelle (<table>). Der Haupttext kommt in die erste Spalte oder in die zweite Spalte, wenn zu dem Element ein Bild angezeigt werden soll. Das Bild kommt dann in die erste Spalte und die Überschrift des Haupttextes erstreckt sich dann über die ersten beiden Spalten.

    Alle weiteren Daten und Steuerelemente erscheinen ab der zweiten bzw. dritten Spalte

    Detailansicht

    Die Detailansicht dient der Darstellung von Daten eines einzelnen Elements. Die Daten werden in Beschreibungslisten <dl> gruppiert. Gruppen werden visuell voneinander getrennt dargestellt.

    Definitionsterme (<dt>) geben den anzuzeigenden Daten einen Namen und können eine detaillierte Beschreibung erhalten, wenn dem Term ein <span> Element hinzugefügt wird. Die detaillierte Beschreibung wird zurückhaltend in einer eigenen Zeile dargestellt.

    Die eigentlichen Daten des darzustellenden Elements werden von einem Definitionsdetail Element(<dd>) umschlossen.

    Zusammengehörende Paare von Definitionstermen und Definitionsdetails müssen von einem Inhaltsteilungselement (<div>) umschlossen.

    Wenn das umschließende Inhaltsteilungselement ein Attribut tabstop=0 besitzt kann das Paar aus Definitionsterm und Definitionsdetail den Fokus erhalten. Der Bereich wird mit der Akzentfarbe umrahmt.

    Formulare

    Formulare werden mit einem form Element umschlossen. Das label umschließt das dazugehörige input Element. Damit kann auf das Attribut for für das Label und id für das Eingabeelemente verzichtet werden.

    Das form Element ist direktes Kind des main Elements. Es zentriert das Formular im Inhaltsbereich und beschränkt die Breite.

    Eingabefelder müssen durch fieldset Elemente gruppiert werden. Fieldset Element erzeugen die abgerundeten Ecken.

    Die Akzentfarbe der Steuerelemente wird durch input {accent-color: var(--color-accent) überschrieben. Mit der Übernahme der Kontrolle über die Akzentfarbe wird eine möglichst konsistente Ansicht gewährleistet. Lediglich komplexe Steuerelemente, die von input[type=date] und ähnlichen Eingabeelementen bereitgestellt werden, passen sich nicht an die gewählte Akzentfarbe an, sondern verwenden (zumindest unter GNOME) die Akzentfarbe des Systemdesigns. Dafür können eigene Elemente mit der Akzentfarbe kontrolliert werden.

    Die Akzentfarbe des Systems kann nicht durch einfache Mittel Browserübergreifend ermittelt werden, so dass eigene Elemente mit dieser Akzentfarbe gestyled werden können.

    Nicht alle Steuerelemente kann die Akzentfarbe zugewiesen werden.

    Das KISS Prinzip hat Vorrang. Das Stylesheet soll möglichst einfach gehalten sein und keine Javascript Unterstützung benötigen.

    Primäre Steuerelemente werden von einem em Element umschlossen.

    Sekundäre Steuerlemente werden nicht besonders ausgezeichnet. Das führt jedoch zu einer Ungereimtheit mit Verknüpfungen, die als Schaltfläche dargestellt werden sollen.

    Steuerelemente und Verknüpfungen mit destruktivem Character werden von einem strong Element umschlossen. Allerdings sollten Verküpfungen keine Destruktiven Aktionen durchführen. Das gehört sich einfach nicht.

    Menüs

    Menüs können an unterschiedlichen Stellen verwendet werden. Im Header als Anwendungsmenü, in der Seitenleiste zur Navigation oder im Inhaltsbereich.

    Menüs in der Kopfzeile (header)

    Menüs in der Kopfzeile dienen Navigation, können aber auch Elemente enthalten, mit denen die aktuelle Ansicht konfiguriert wird und zum Beispiel zwischen Listen- und Rasteransicht oder die Sortierung umgeschaltet.
